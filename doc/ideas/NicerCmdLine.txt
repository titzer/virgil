Idea: better command line experience to improve developer ergonomics

Background:

1. Too many platform-specific scripts that need expert knowledge to use.

v3c -heap-size=XXmb
v3c-x86-linux-nogc
v3c-wasm-wave

v3i
v3c

2. Don't like the verbosity of a potential "virgil compile" and "virgil build", etc

3. Like the terseness of "v3c"

=> Make v3c a script and leave "Aeneas" as the raw compiler command?

----------------------------------
Command-line examples

% v3i MyProgram.v3 args
  runs program with {args}
  
% v3c MyProgram.v3
  compiles program to host and makes runnable ./MyProgram
  ? does incremental, timestamp-based build

% v3c --run program.v3
  compiles program to host and runs MyProgram
  ? does incremental build

% v3c --no-gc program.v3
% v3c --heap-size=5m program.v3
% v3c --gc=genms program.v3
% v3c --target=x86 program.v3
% v3c --target=m32 program.v3
% v3c --analyze program.v3
% v3c --check program.v3
