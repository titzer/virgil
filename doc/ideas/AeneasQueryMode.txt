Idea: Aeneas compiler supports "query" mode

This allows a user (or agent) to load, initialize, and compile a program to a target,
after which the compiler enters into a REPL which allows query commands for many different
kinds of information about a program and its compilation.

Queries:
def <id>        shows definition(s) of identifier, such as a class, type, method, var, etc
read <id>       shows reads of a field or method
write <id>      shows writes of a field
use <id>        shows uses (reads/writes) of a definition
val <id>        shows inferred constant value / value range of definitions
type <id>       shows inferred type for identifier
ast <id>        shows abstract syntax tree of a given method
ssa <id>        shows SSA of a given method
mach <id>       shows machine representation of a given method
ra <id>         show reachability information of a given type, method or field
layout <id>     shows memory layout of a given class or ADT
eval <expr>     evaluates top-level expression (side effects discarded)

Example: 

v3c -target=x86-linux -query MyFile1.v3 MyFile2.v3
% def MyClass
MyFile1.v3 @ 13:1
% read MyClass.foo
MyFile1.v3 @ 14:12, 15:2
MyFile2.v3 @ 101:99
% use MyClass.foo
MyFile1.v3 @ 14:12, 15:2
MyFile2.v3 @ 101:99
% val MyClass.foo
def int = 99
